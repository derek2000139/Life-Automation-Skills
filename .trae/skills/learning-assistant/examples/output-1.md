# 闪卡集 · JavaScript 闭包

## Card 1 · 基础
**Q: 闭包是什么？用一句话描述。**
<details><summary>查看答案</summary>
闭包是一个函数，它能记住并访问它被创建时所在的词法作用域，
即使该函数在其词法作用域之外执行。
</details>

## Card 2 · 基础
**Q: 以下代码输出什么？为什么？**
```javascript
function outer() {
  let count = 0;
  return function() { count++; return count; }
}
const fn = outer();
console.log(fn()); // ?
console.log(fn()); // ?
```
<details><summary>查看答案</summary>
输出 1，然后 2。
因为 fn 通过闭包保持了对 outer 函数内 count 变量的引用，
每次调用 fn，count 都在原来的基础上自增，而不是重置。
</details>

## Card 3 · 进阶
**Q: 闭包和普通函数在内存上的区别是什么？**
<details><summary>查看答案</summary>
普通函数执行完毕后，其作用域内的变量会被垃圾回收。
闭包会阻止这个过程，因为内部函数持有对外部变量的引用，
导致外部函数的作用域无法被释放。滥用闭包可能导致内存泄漏。
</details>

# 费曼学习单 · JavaScript 闭包

## 用大白话解释
你开了一家咖啡店（外部函数），雇了一个员工（内部函数）。
员工下班后离开了店面（函数执行结束），
但他的脑子里还记得今天的收银机密码（变量 count）。
第二天你让他去取钱，他不需要重新被告知密码，因为他还记着。
这就是闭包——函数"记住"了它出生时的环境。

## 生活类比
> 就像你出国留学，离开了家，但你还记得家里的 WiFi 密码（词法作用域的变量）。你虽然在国外（函数在作用域外执行），但密码还是可以用。

## 最容易搞错的地方
⚠️ 闭包保存的是**变量的引用**，不是变量的值的副本。
经典错误：在循环里用 var 创建闭包，所有闭包都指向同一个变量的最终值。
